uv — как оно устроено

Схема резолвера, lock-файла и синка окружения для проекта itmo-gp1. Редактируй блоки <text> в SVG или HTML-секциях.

1. Общий поток: pyproject.toml → uv.lock → .venv

INPUTS — ручные источники истины RESOLVER + lock — машинный артефакт SYNC + .venv — материализация --- пунктир — uv run запускает lazy-sync

2. Слои зависимостей в pyproject.toml

[project].dependencies

Всегда ставятся. Все продакшен-импорты проекта.

dependencies = [
  "torch==2.9.1",
  "pyctcdecode>=0.5.0",
  "num2words>=0.5.13",
]

default попадают в любой uv sync.

[project.optional-dependencies] (extras)

Опциональные «набор фич», стандарт PEP 621. Видны при pip install itmo-gp1[bpe].

[project.optional-dependencies]
bpe = ["sentencepiece>=0.2.0"]

Ставятся только при --extra bpe / --all-extras. off by default

[dependency-groups] (PEP 735)

uv-родной способ. Удобнее extras для dev/lm/test — не тянутся когда кто-то ставит пакет как зависимость.

[dependency-groups]
dev = ["pytest", "ipykernel", "pandas"]
lm  = ["kenlm>=0.2.0"]

--group lm включит, --no-group lm выключит. группу можно сделать default

[tool.uv] default-groups

Какие dependency-groups попадают в uv sync без флагов.

[tool.uv]
default-groups = ["dev", "lm"]

У нас dev + lm → kenlm ставится каждым uv sync. Убрать в CI: --no-group lm.

[tool.uv.sources] + [[tool.uv.index]]

Переопределяют откуда брать пакет, с маркерами платформы.

[[tool.uv.index]]
name = "pytorch-cu128"
url  = "https://download.pytorch.org/whl/cu128"
explicit = true   # не использовать кроме явных ссылок

[tool.uv.sources]
torch      = [{ index = "pytorch-cu128",
                marker = "sys_platform == 'linux'" }]
torchaudio = [{ index = "pytorch-cu128",
                marker = "sys_platform == 'linux'" }]

На Linux резолвер берёт torch с cu128-индекса (Blackwell sm_120), на macOS — с PyPI.

[tool.uv] conflicts

Декларация, что extras/groups несовместимы. Резолвер строит разные решения в одном lock.

[tool.uv]
conflicts = [
  [{ extra = "train" }, { extra = "lm" }]
]

Нужно только если реально есть конфликт (разные numpy и т.п.). У нас сейчас не используется.

3. Ключевые команды и что они делают

КомандаЧто делаетПобочка
uv lock Только решает зависимости и пишет uv.lock. Не трогает venv. safe
uv sync Материализует проекцию lock в venv. Проекция = dependencies + default-groups + установленные --extra/--group. удаляет лишнее
uv sync --extra lm Расширяет проекцию указанным extra. safe
uv sync --no-group lm Исключает группу, даже если она в default-groups. safe
uv sync --inexact Не удалять пакеты, которых нет в проекции (ручные pip install выживут). "грязный" venv
uv sync --frozen Не перерешивать, даже если pyproject новее — пойдёт строго по lock. CI-стандарт. reproducible
uv add <pkg> Редактирует pyproject.toml + перелочивает + синкает. Эквивалент pip install -e . + edit. safe
uv add --group lm kenlm Добавляет пакет в конкретную группу. safe
uv run <cmd> Прогоняет команду внутри managed venv, делая lazy-sync при необходимости. safe
uv pip install <pkg> Ставит в текущий venv в обход lock. Всё, что поставлено так — будет снесено следующим uv sync. смертник
UV_PROJECT_ENVIRONMENT=.venv-lm uv sync Ставит синк во второй venv в этой же папке проекта. Удобно для двух несовместимых стеков. safe

4. Проекция lock → venv: как собирается итоговый набор пакетов

projection = dependencies
           ∪ (optional-deps[e] for e in extras_active)
           ∪ (dependency-groups[g] for g in groups_active)

groups_active = (default-groups ∪ --group) \ --no-group

→ resolve_from_lock(projection)    # выбирает конкретные wheels из uv.lock
→ install(set)                     # добавляет то, чего нет
→ remove(venv \ set)                # ⚠ удаляет всё лишнее (если не --inexact)

Именно шаг «remove» и съедал kenlm / pandas — их не было в активной проекции.

5. Два окружения в одном проекте

# train-окружение: всё кроме lm
uv sync --no-group lm

# lm-окружение (инференс с KenLM) — отдельный venv
UV_PROJECT_ENVIRONMENT=.venv-lm uv sync --group lm

# jupyter kernels
UV_PROJECT_ENVIRONMENT=.venv     uv run python -m ipykernel install --user --name gp1
UV_PROJECT_ENVIRONMENT=.venv-lm  uv run python -m ipykernel install --user --name gp1-lm

Сейчас не нужно — один venv справляется. Оставить как рецепт на случай, если torch ≥ 2.11 реально потребует иного numpy, чем kenlm.

6. Шпаргалка по файлу нашего проекта

pyproject.toml
├── [project].dependencies            # torch==2.9.1 (cu128), torchaudio, pyctcdecode, …
├── [project.optional-dependencies]
│   └── bpe = [sentencepiece]         # uv sync --extra bpe
├── [dependency-groups]
│   ├── dev = [pytest, ipykernel, pandas, …]
│   └── lm  = [kenlm]
├── [tool.uv]
│   └── default-groups = ["dev", "lm"]   # → goes in every `uv sync`
├── [[tool.uv.index]] pytorch-cu128   # Blackwell sm_120 wheels
└── [tool.uv.sources] torch, torchaudio → pytorch-cu128 (только linux)

Файл можно редактировать в любом редакторе. Все цвета и тексты — inline. Перерисовать SVG — меняй координаты x/y у <rect>/<text>/<path>.
