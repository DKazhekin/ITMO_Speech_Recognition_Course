1. Опорный пример и форма батча

Чтобы видеть, как именно меняется T (временная ось), берём один сэмпл длиной 2 секунды речи — реальный средний размер для задачи "русские цифры":

Сырое аудио
[B=1, T_audio=32000] float32, 16 kHz моно
n_fft
512 (frequency bins = n_fft/2 + 1 = 257)
hop_length
160 (10 ms шаг)
win_length
400 (25 ms окно Ханна)
n_mels
80
Количество mel-кадров
T_mel = T_audio / hop + 1 = 32000/160 + 1 = 201
В батче: длина T каждого сэмпла разная, но после collate_fn все сэмплы паддятся нулями до T_max в батче. Модель получает [B, 80, T_max] и mel_lengths: [B] — фактические длины до padding. Все shape-формулы ниже работают и для батча — просто на месте T подставь максимум по батчу.

Количество целевых символов U для "сто пятьдесят девять тысяч четыреста семьдесят три" (число 159473) — около 44 символов (с пробелами). Инвариант CTC: T' >= 2 · U_max, где T' — выход энкодера.

Файл: src/gp1/features/melbanks.py. Общий для всех 4 моделей, применяется внутри Trainer._mel_features() до модели.

Почему log-mel, а не сырой спектр: слух человека воспринимает интенсивность логарифмически, и log-mel стабилизирует динамический диапазон. n_mels=80 — стандартный для ASR баланс между частотным разрешением и размером фичи.

3. QuartzNet 10x4 — TCS-conv стек

Идея

Стек из 10 блоков × 4 подблока каждый, где каждый подблок — Time-Channel-Separable свёртка (depthwise 1D + pointwise 1×1 + BN + ReLU + Dropout). TCS даёт почти качество обычной свёртки за четверть параметров. Единственное понижение временной оси — в прологе (stride=2). Нет attention, нет RNN — чистый CTC-friendly 1D-stack.

Бюджет: ~4.35M параметров. Subsample factor: 2 (жёстко).

Пошаговая трассировка

[1, 256, 101]
[4]

[1, 256, 101]

Tap: intermediate = transpose(B2_out)
[1, 101, 256]

Что такое "intermediate" и зачем он

На середине глубины (после B2, 40% пути) мы достаём промежуточное представление. Оно уже кодирует акустику, но ещё не "дошло" до финальной классификации. Для базового обучения эта промежуточка не используется — её видит только InterCTC-голова в эксперименте 01a (секция 8.1 ниже).

Что делает один TCSConv-подблок

x              : [B, C=256, T=101]
  ├─ depthwise Conv1d (C → C, kernel=33, groups=C)  ← обрабатывает только время, канал в канал
  │   → [B, 256, 101]
  ├─ pointwise Conv1d (C → C, kernel=1)             ← микширует каналы, не трогает время
  │   → [B, 256, 101]
  ├─ BatchNorm1d
  ├─ (+ residual если c_in == c_out и stride == 1)
  ├─ ReLU
  └─ Dropout
  → [B, 256, 101]

Ключ: depthwise отделяет работу "по времени" от работы "по каналам" — parameters = k·C + C² вместо k·C·C у обычной Conv1d.

4. CRDNN — Conv2D + BiGRU

Идея

Две 2D-свёртки на (freq, time) → разворачиваем в [B, T, feat] → BiGRU → линейная голова. Conv2D-фронтенд сжимает частотную ось вдвое дважды (80 → 40 → 20), не трогая время (при subsample_factor=1). BiGRU моделирует долговременные зависимости через рекуррентность в обе стороны.

Бюджет: ~4.9M параметров при d_cnn=64, rnn_hidden=256, rnn_layers=2. Subsample factor: 1 или 2.

Пошаговая трассировка (subsample_factor=1)

[1]
log-mel (вход)
[1, 80, 201]
[2]
unsqueeze(1) — добавляем канал
[1, 1, 80, 201]
[3]
Conv2d(1→64, k=3, stride=(2,1)) + BN + ReLU + Dropout
[1, 64, 40, 201]
[4]
Conv2d(64→64, k=3, stride=(2,1)) + BN + ReLU + Dropout
[1, 64, 20, 201]
[5]
permute(0,3,1,2).reshape — flatten freq×channels
[1, 201, 1280]
[6]
BiGRU(1280 → 256×2, 2 слоя, bidirectional)
[1, 201, 512]
[7]
LayerNorm + Dropout
[1, 201, 512]
[8]
Linear(512 → 35)
[1, 201, 35]
[9]
.float() + log_softmax(dim=-1)
[1, 201, 35]
T' = ceil(T_mel / subsample_factor); при s=1: T' == T_mel
Почему Conv2D, а не 1D: трактовка (freq, time) как 2D-картинки позволяет свёртке "смотреть" сразу в локальное окно по обеим осям. Classic SpeechBrain/ESPnet паттерн. При этом время не трогаем — stride только по частоте. BiGRU не любит короткие последовательности и даёт лучший effective receptive field при полном T.

Что делает BiGRU

GRU — gated recurrent unit. На каждом шаге t принимает x_t и предыдущее состояние h_{t-1}, через update-gate и reset-gate вычисляет новое h_t. Bidirectional = два GRU (прямой и обратный), их выходы конкатенируются по каналам. Итого: каждый фрейм на выходе видит весь контекст до и после себя.

5. Efficient Conformer — progressive downsample

Идея

Conformer (macaron-FFN → MHSA → Conv module → macaron-FFN) — стандарт современного ASR. Обычный Conformer дорог: self-attention стоит O(T²). Efficient Conformer (Burchi & Vielzeuf 2021) делит блоки на стадии разной временной дискретизации: stage 1 работает на T/2, между stage 1 и stage 2 — downsample /2, stages 2-3 работают на T/4. Итого attention-cost в 4× меньше, чем "в лоб".

Бюджет: ~4.2M параметров. Subsample factor: 4 (жёстко, 2×2).

Пошаговая трассировка

[1]
log-mel (вход)
[1, 80, 201]
[2]
Prologue TCSConv(80→64, k=11, stride=2)
[1, 64, 101]
[3]
input_conv 1×1 (64→96) + transpose + LayerNorm
[1, 101, 96]
[4]
Stage 1: 4× ConformerBlock(d=96, heads=4, k=15)
[1, 101, 96]
[5]
DownsampleBlock — stride-2 Conv1d, 96→128
[1, 51, 128]
[6]
Stage 2: 4× ConformerBlock(d=128, heads=4, k=15)
[1, 51, 128]
[7]
Stage 3: 4× ConformerBlock(d=128, heads=4, k=15)
[1, 51, 128]
[8]
Linear(128 → 35)
[1, 51, 35]
[9]
.float() + log_softmax(dim=-1)
[1, 51, 35]
T' = ceil(ceil(T_mel / 2) / 2) = ceil(201/2)/2 = 51

Анатомия одного ConformerBlock

Macaron-pattern: две "половинки" FFN с весом 0.5 обрамляют центральные MHSA и Conv module. Все подмодули — с pre-norm и residual connection.

x : [B, T, d_model]
  │
  ├─ x = x + 0.5 · FFN1(LayerNorm(x))                    ← macaron half-FFN
  │        FFN: Linear(d → d·4) → SiLU → Dropout → Linear(d·4 → d) → Dropout
  │
  ├─ y = LayerNorm(x)
  │     attn_out, _ = MultiheadAttention(y, y, y)        ← self-attention
  │     x = x + Dropout(attn_out)
  │
  ├─ x = x + ConvModule(x)                                ← локальный контекст
  │        LayerNorm → pointwise(d → 2d) → GLU → depthwise(k=15) → BN → SiLU → pointwise(2d → d)
  │
  ├─ x = x + 0.5 · FFN2(LayerNorm(x))                    ← вторая macaron half-FFN
  │
  └─ LayerNorm(x)
  → [B, T, d_model]

Зачем так много компонентов: MHSA ловит глобальные зависимости, ConvModule — локальный контекст, FFN'ы — нелинейную комбинацию признаков. Macaron-паттерн (половина FFN до/после) эмпирически лучше чем одна полная FFN.

6. Fast Conformer BPE — subsample ×4 → 16 блоков

Идея

Rekesh et al. (2023). Всё просто: один SubsampleConv в начале сжимает время ×4, дальше идёт стек из 16 одинаковых ConformerBlock (без стадий, без промежуточных downsampling). Вся модель работает на T/4 фреймах. Единственная в проекте модель на BPE-vocab (256 subwords).

Бюджет: ~4.2M параметров. Subsample factor: 4 (жёстко).

Пошаговая трассировка

[1]
log-mel (вход)
[1, 80, 201]
[2]
SubsampleConv factor=4 (две Conv2d stride=2 + Linear 1920→96)
[1, 51, 96]
[3]
Dropout(0.1)
[1, 51, 96]
[4]
16× ConformerBlock(d=96, heads=4, k=9)
[1, 51, 96]
[5]
Linear(96 → 257) — vocab = 256 BPE + 1 blank
[1, 51, 257]
[6]
.float() + log_softmax(dim=-1)
[1, 51, 257]
T' = ceil(ceil(T_mel / 2) / 2) = 51; Vocab = 257 (BPE-256 + blank)

Что делает SubsampleConv factor=4

mel : [1, 80, 201]
  unsqueeze(1)          → [1, 1, 80, 201]        # добавляем канал
  Conv2d(1→96, k=3, s=2)→ [1, 96, 40, 101]       # stride 2 по обеим осям
  ReLU
  Conv2d(96→96, k=3, s=2)→ [1, 96, 20, 51]       # stride 2 по обеим
  ReLU
  permute+reshape       → [1, 51, 96·20=1920]    # flatten (freq × channels)
  Linear(1920 → 96)     → [1, 51, 96]            # проекция в d_model

Зачем BPE-vocab

Для char-модели целевой токен — одна русская буква. Для BPE-256 — subword (часто целая морфема или даже короткое слово): "тысяч", "▁три", "▁сто" могут быть единым токеном. BPE-последовательность в 2-3× короче char-последовательности для того же текста, что даёт запас T'/U_max при тех же subsample=4. Decoding: BPEVocab.decode(ids) → text → words_to_digits(text).

7. Как работает CTC-loss

Проблема: модель выдаёт T' кадров, а таргет — U символов, и алайнмент между ними неизвестен. У Russian "999999" целых 44 символа, а у 2-секундного аудио только 51 выход.

Решение CTC: добавляем специальный blank-токен, разрешаем повторы символов, и усредняем по всем валидным алайнментам автоматически. В итоге loss равен -log(P(target | audio)), где P — сумма по всем путям (алгоритм forward-backward).

loss = -log(P(y | x))
P(y | x) = Σ_{π ∈ A(y)} P(π | x), где A(y) — все CTC-алайнменты для target y

Инвариант: T' >= 2 · U_max. Если T' меньше, алгоритм forward-backward возвращает -inf — модель никогда не научится. Отсюда жёсткие ограничения на subsample_factor в каждой модели.

Fp32 island: в Trainer forward модели идёт в autocast(dtype=fp16) для скорости, но вызов ctc_loss(log_probs.float(), ...) обёрнут в autocast(enabled=False). Причина: CTC forward-backward в fp16 нестабилен, даёт NaN на длинных сэмплах.

8. Эксперименты: InterCTC, CR-CTC, WordAux

Три техники, которые можно опционально подключать к основному CTC-обучению. Все три живут inline в ноутбуках notebooks/experiments/, не в src/ — потому что каждая требует специфичного изменения training loop. Общая идея одна: добавить вторую loss с весом, чтобы регуляризовать основную CTC-задачу.

8.1. InterCTC — вспомогательная CTC на промежуточном слое

Что

Поставить вторую CTC-loss на представления с середины энкодера, не только на финальном выходе. Заставляем внутренние слои тоже "знать" о целевом тексте, а не только последний слой.

Paper: Lee & Watanabe 2020 "Intermediate Loss Regularization for CTC-Based Speech Recognition".

Где подключается в QuartzNet

У QuartzNet есть tap после блока B2 — он экспонируется как EncoderOutput.intermediate формы [B, 101, 256]. Ноутбук 01a_quartznet_inter_ctc.ipynb определяет inline-класс:

class InterCTCHead(nn.Module):
    def __init__(self, d_mid: int, vocab_size: int, blank_id: int = 0):
        super().__init__()
        self.proj = nn.Linear(d_mid, vocab_size)  # 256 → 35
        self.ctc = nn.CTCLoss(blank=blank_id, reduction="mean", zero_infinity=True)

    def forward(self, mid_features, input_lengths, targets, target_lengths):
        logits = self.proj(mid_features.float())         # [B, T', V]
        log_probs = F.log_softmax(logits, dim=-1)
        return self.ctc(log_probs.transpose(0,1), targets, input_lengths, target_lengths)

Как меняется training loop

[1]
out = model(mel, mel_lengths) → out.log_probs + out.intermediate
logp: [B, 101, 35]
inter: [B, 101, 256]
[2]
loss_main = ctc(log_probs, targets, …)
scalar
[3]
loss_inter = inter_head(out.intermediate, …)
scalar
[4]
loss = loss_main + weight · loss_inter (weight=0.3)
scalar
[5]
loss.backward()
—

Почему работает: градиент от второй CTC усиливает обучение нижних слоёв и действует как регуляризация — модель не может "откладывать всю работу" на последние блоки.

8.2. CR-CTC — consistency regularization

Что

Пропустить один и тот же сэмпл через модель дважды с разными аугментациями. Модель должна давать похожие распределения над токенами — штрафуем за расхождение через симметричный KL.

Суть: "Твоё предсказание не должно зависеть от того, какой SpecAug-маской я тебя накрыл".

Математика

loss_cr = ½ · [ KL(p_a || p_b) + KL(p_b || p_a) ]
где p_a, p_b — log_probs для view 1 и view 2 одного и того же аудио
(с маскированием по input_lengths и min_prob floor для численной стабильности)

Где подключается

Ноутбук 01c_quartznet_cr_ctc.ipynb. Упрощённо view 2 делается на mel-уровне вторым SpecAugmenter с другим seed:

[1]
mel_v1 = spec_aug1(mel_extractor(audio), lengths)
[B, 80, T_mel]
[2]
mel_v2 = spec_aug2(mel_extractor(audio), lengths) ← другая маска
[B, 80, T_mel]
[3]
out_a = model(mel_v1, …)
logp: [B, T', V]
[4]
out_b = model(mel_v2, …)
logp: [B, T', V]
[5]
loss_main_a = ctc(out_a, targets, …)
loss_main_b = ctc(out_b, targets, …)
scalars
[6]
loss_cr = CRCTCLoss()(out_a.log_probs, out_b.log_probs, out_a.output_lengths)
scalar
[7]
loss = ½(loss_main_a + loss_main_b) + weight · loss_cr
scalar
[8]
loss.backward()
—

Почему работает: регуляризация смягчает переобучение на конкретных аугментационных паттернах. Модель учится устойчивому распределению вместо запоминания тренировочного набора масок.

8.3. WordAux — вспомогательная CTC на уровне слов

Что

Параллельно с основной char-CTC (выход в символы) добавить вторую CTC-голову, которая предсказывает номерные слова русского ("один", "два", ..., "тысяча"). Итого модель учится одновременно воспроизводить буквы и слова.

Паттерн "auxiliary target" из multi-task learning — приближает к нужной семантике (слова-числительные ограничены NUMBER_WORDS, 42 штуки).

Ingredient 1: inline WordVocab

Закрытый словарь из 42 русских числительных: "ноль", "один", "одна", "два", "две", ..., "тысяча", "тысячи", "тысяч". Blank = id 0, остальные 1..42.

Ingredient 2: inline _encode_words

def _encode_words(trans_list, wvocab):
    encoded = [wvocab.encode(digits_to_words(t)) for t in trans_list]
    # "139473" → "сто тридцать девять тысяч четыреста семьдесят три"
    #         → [id_сто, id_тридцать, id_девять, id_тысяч, id_четыреста, id_семьдесят, id_три]
    lengths = torch.tensor([len(e) for e in encoded], dtype=torch.int64)
    U = max(lengths.max().item(), 1)
    tensor = torch.zeros(len(encoded), U, dtype=torch.int64)
    for i, e in enumerate(encoded):
        tensor[i, :len(e)] = torch.tensor(e, dtype=torch.int64)
    return tensor, lengths

Ingredient 3: WordAuxCTCHead

class WordAuxCTCHead(nn.Module):
    def __init__(self, d_enc: int, word_vocab_size: int, blank_id: int = 0):
        super().__init__()
        self.proj = nn.Linear(d_enc, word_vocab_size)  # 256 → 43
        self.ctc = nn.CTCLoss(blank=blank_id, reduction="mean", zero_infinity=True)

    def forward(self, enc_features, input_lengths, word_targets, word_tlens):
        logits = self.proj(enc_features.float())
        log_probs = F.log_softmax(logits, dim=-1)
        return self.ctc(log_probs.transpose(0,1), word_targets, input_lengths, word_tlens)

Как меняется training loop

[1]
out = model(mel, mel_lengths)
logp: [B, 101, 35]
inter: [B, 101, 256]
[2]
loss_main = ctc(out.log_probs, char_targets, …)
scalar
[3]
word_t, word_len = _encode_words(batch.transcriptions, word_vocab)
[B, U_w], [B]
[4]
loss_word = word_head(out.intermediate, out.output_lengths, word_t, word_len)
scalar
[5]
loss = loss_main + weight · loss_word
scalar
[6]
loss.backward()
—

Ключевое отличие от InterCTC: обе головы пишут в разные словари. Char-head = 35 букв, Word-head = 43 числительных. При inference используется только char-head.

Общий шаблон всех трёх экспериментов

loss_total = loss_main_ctc + λ · loss_auxiliary
↓ .backward()
обе головы обновляются одновременно, но при inference используется только main

λ (вес вспомогательной loss) — гиперпараметр. Типичные значения: 0.1 – 0.5

Fast Conformer BPE ×4 BPE-257 [1, 51, 257] SubsampleConv /4 + 16× ConfBlock@96, k=9 AdamW + Noam warmup

Формулы выходной длины

QuartzNet 10x4      : T' = ceil(T_mel / 2)             = (T_mel + 1) // 2
CRDNN (s=1)         : T' = T_mel                       # время не трогаем
CRDNN (s=2)         : T' = ceil(T_mel / 2)             = (T_mel + 1) // 2
Efficient Conformer : T' = ceil(ceil(T_mel / 2) / 2)   # два stride-2 шага
Fast Conformer BPE  : T' = ceil(ceil(T_mel / 2) / 2)   # SubsampleConv = 2 × stride-2
